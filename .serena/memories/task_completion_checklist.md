# タスク完了時のチェックリスト

## 1. コード品質チェック

### TypeScriptコンパイルエラー確認
```bash
npm run build
```
- エラーがないことを確認
- 型エラーを解決

### 開発サーバーでの動作確認
```bash
npm run dev
```
- サーバーが正常に起動することを確認
- エラーログがないか確認

## 2. データベース関連

### Prismaマイグレーション
スキーマを変更した場合:
```bash
npx prisma migrate dev --name <変更内容>
npx prisma generate
```

## 3. コードレビュー項目

### コーディング規約
- [ ] 命名規則に従っているか
- [ ] インポート文に `.js` 拡張子が付いているか
- [ ] 適切な型定義がされているか
- [ ] 未使用の変数/パラメータがないか
- [ ] 適切なエラーハンドリングがあるか

### 機能チェック
- [ ] 追加/変更した機能が正常に動作するか
- [ ] エッジケースを考慮しているか
- [ ] エラー時の挙動が適切か

## 4. Git操作

### コミット前
```bash
git status              # 変更ファイル確認
git diff                # 差分確認
git add <files>         # ステージング
git commit -m "明確な説明" # コミット
```

### プッシュ前
- コミットメッセージが明確か確認
- 不要なファイルが含まれていないか確認

## 5. ドキュメント更新
- 新しいエンドポイントを追加した場合、APIドキュメントを更新
- 大きな変更の場合、DEPLOYMENT.mdや関連ドキュメントを更新
